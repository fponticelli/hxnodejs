package js.node

typedef js.node.HttpRequestOptions

defined in js.node.Http

Available on all platforms

Type of the options object passed to Http.request.

Properties

socketPath:Null<String>

Unix Domain Socket (use one of host:port or socketPath)

port:Null<Int>

Port of remote server. Defaults to 80.

path:Null<String>

Request path. Defaults to '/'. Should include query string if any. E.G. '/index.html?page=12'

method:Null<Method>

A string specifying the HTTP request method. Defaults to 'GET'.

localAddress:Null<String>

Local interface to bind for network connections.

hostname:Null<String>

To support Url.parse hostname is preferred over host

host:Null<String>

A domain name or IP address of the server to issue the request to. Defaults to 'localhost'.

headers:Null<DynamicAccess<String>>

An object containing request headers.

There are a few special headers that should be noted:

Sending a 'Connection: keep-alive' will notify Node that the connection to the server
should be persisted until the next request.

Sending a 'Content-length' header will disable the default chunked encoding.

Sending an 'Expect' header will immediately send the request headers.
Usually, when sending 'Expect: 100-continue', you should both set a timeout
and listen for the continue event. See RFC2616 Section 8.2.3 for more information.

Sending an Authorization header will override using the auth option to compute basic authentication.

auth:Null<String>

Basic authentication i.e. 'user:password' to compute an Authorization header.

agent:Null<EitherType<Agent, Bool>>

Controls Agent behavior. When an Agent is used request will default to Connection: keep-alive.

Possible values:

null (default): use global `Agent` for this `host` and `port`.
`Agent` object: explicitly use the passed in `Agent`.
false: opts out of connection pooling with an `Agent`, defaults request to 'Connection: close'.
